Blindstore

Blindstore is a private information retrieval data store. It provides an online lookup service which returns the correct answer without ever knowing what the question was.

Centralised

Unlike a distributed hash table, Blindstore is not prone to a Sybil attack. The data is not inherently public, since the server is free to restrict access with authentication and authorization.

Privacy

The protocol guarantees that the server can't know which record the client requested. The client still only receives the record they requested.

Here be dragons

Blindstore is very much work in progress, made using experimental cryptographic software. For the moment, it is for research purposes only.

Protocol

Implementation

Blindstore implements a cryptographic protocol called Strong Private Information Retrieval or Oblivious Transfer, specifically, the one defined in Single-Database Private Information Retrieval from Fully Homomorphic Encryption by X. Yi, M.G. Kaosar, R. Paulet, and E. Bertino.

In the Private Information Retrieval protocol, there's a server (sender) which holds some bits of information, and a client (receiver) which wants to retrieve some of the bits in such a way that the server won't know which bits the client wanted.

Model

Using the Blindstore protocol, a client can assume that the server is not trustworthy.

The client locally encrypts the index of the required record using their FHE public key, and sends it to the server. The server evaluates certain homomorphic functions over the encrypted index and the data, and sends an encrypted response. Using their private key, the client can decrypt it to obtain the plaintext response.

Note that verification of the response's authenticity is out of scope of the Blindstore protocol.

Homomorphism in this case is a property of asymmetric encryption which says there exist certain homomorphic functions \(f\) such that: \[\text{DECRYPT}_{sk}(f\ (\text{ENCRYPT}_{pk}(\text{m}))) = f(\text{m})\]

Feasibility

The total communication and computation cost per request are roughly logarithmic relative to the database size.

Privacy comes at the cost of this computation and communication overhead. With the current implementation it takes seconds to generate a response even for small datasets (hundreds of records). Thankfully, the response generation algorithm is highly parallelisable, and we're working on making it significantly faster.

The communication cost per request is \(O(\gamma\log (m \cdot L))\), and the computation cost per request is \(O(\gamma\log m + \gamma L)\), where \(m\) is the number of records in the database, \(L\) the size of one record, and \(\gamma\) the ciphertext size.

Applications

Blindstore can be used as a building block for different privacy-sensitive applications where the server is considered not trustworthy, such as:

  • Private domain name resolution
  • Domain name availability search
  • Private peer-to-peer signaling
